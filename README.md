# Ollama Service Manager

v 1.1s

Легковесная утилита с графическим интерфейсом (GUI) для macOS, предназначенная для управления локальным сервером Ollama. Позволяет запускать, останавливать и мониторить демон, а также управлять библиотекой моделей без использования терминала.

## Особенности
- **Полный контроль демона:** Запуск с корректной сетевой привязкой (`0.0.0.0:11434`) и принудительным отключением облачных запросов (`OLLAMA_NO_CLOUD=true`).
- **Автоматическая настройка среды:** Решает проблему отсутствия `PATH` в GUI-приложениях macOS. Автоматически находит бинарник Ollama в `/Applications`, Homebrew или системных путях.
- **Мониторинг логов:** Вывод `stdout/stderr` демона в реальном времени. Поддержка выделения, копирования (`Cmd+C`) и открытия файла лога в системном редакторе.
- **Управление моделями:** Отдельное окно для просмотра установленных моделей, загрузки новых (`ollama pull`) с визуализацией прогресса и удаления (`ollama rm`).
- **Портативность:** Динамическое определение путей к логам и иконкам. Корректно работает как из исходников, так и из скомпилированного `.app`.
- **Чистый стек:** Написан на стандартной библиотеке Python (`tkinter`, `subprocess`, `threading`). Не требует установки тяжелых фреймворков.

## Требования
- macOS (Intel или Apple Silicon)
- Python 3.9 или выше
- Установленная Ollama (официальный `.app` или установка через `brew`)

## Запуск из исходников
1. Клонируйте или скачайте репозиторий.
2. Убедитесь, что у вас установлен Python 3.
3. Запустите менеджер:
```bash
python3 ollama_manager.py
```

## Сборка в приложение (.app)
Для создания автономного macOS-приложения используется `PyInstaller`.

1. Установите сборщик:
```bash
pip3 install pyinstaller
```
2. Выполните сборку (находясь в корне репозитория):
```bash
pyinstaller --windowed --icon=icon.icns --name "Ollama Service" ollama_manager.py
```
3. Готовое приложение появится в директории `dist/`. Файл `Ollama Service.app` можно переместить в `/Applications` или запустить с рабочего стола.
*Примечание: Файл `ollama_daemon.log` будет создаваться в папке `Contents/MacOS/` внутри пакета.*

## Устранение неполадок
- **Ошибка `bind: address already in use`:** Приложение автоматически пытается завершить фоновые процессы `ollama serve` перед запуском. Если ошибка сохраняется, убедитесь, что стандартная Ollama из меню macOS остановлена.
- **Модели не отображаются:** Проверьте, что Ollama корректно установлена и доступна по пути `/Applications/Ollama.app/Contents/Resources/ollama`.
- **Логи не пишутся:** При запуске `.app` из системной папки `/Applications` у приложения могут не быть прав на запись. Переместите `.app` в пользовательскую директорию (например, `~/Documents` или Рабочий стол).
